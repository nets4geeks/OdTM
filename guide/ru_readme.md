
# Методические рекомендации по онтологическому моделированию угроз

Данные рекомендации описывают как использовать методику моделирования угроз облачным компьютерным системам, основанную на онтологическом анализе и соответствующем
[каталоге шаблонов угроз](https://nets4geeks.github.io/acctp/).

Методика моделирования использует графическую нотацию диаграмм потоков данных (DFD - Data Flow Diagram) для описания информационных потоков между 
компонентами систем и внешними сущностями. Для представления предметной области облачных компьютерных систем используется онтологическая предметно-ориентированная модель,
полученная из каталога шаблонов угроз.

Предложенные средства позволяют автоматически создавать модели угроз (списки возможных угроз) для типичных облачных конфигураций.
Подобные списки угроз могут быть использованы разработчиками для анализа проблем облачных компонентов и внедрения безопасных архитектурных решений.

Рекомендации могут быть полезны для тех, кто планирует внедрять облачные технологии и не является экспертом в информационной безопасности.

Рекомендации оформлены в виде набора лабораторных работ, которые могут быть легко адаптированы для образовательных целей.

Вы можете использовать стороннее приложение ([OWASP Threat Dragon](https://owasp.org/www-project-threat-dragon/)) для просмотра диаграмм потоков данных
и анализа результатов автоматического моделирования угроз.

Непосредственно для создания моделей угроз была использована утилита [OdTMServer](../applications/OdTMServer), разработанная в рамках данного проекта.
